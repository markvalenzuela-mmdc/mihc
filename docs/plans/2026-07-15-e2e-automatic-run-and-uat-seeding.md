# E2E Automatic Run and UAT Profile Seeding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking: - [ ].

**Goal:** Reuse the E2E step mocker for UAT-compatible profile seeding, limit the workspace selector to step 1, and publish authenticated E2E and smoke-test requests to Inngest.

**Architecture:** Keep profile IDs, statuses, and flow types as seed scenario metadata while generating profile names, emails, and complete form data through the existing getE2eProfileStepMockValues utility. Add a focused E2E request action that publishes a new e2e-test/requested event, and authenticate both request actions server-side so the current user ID is the event attribution.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript, TanStack Query/Form, Drizzle, Inngest, Faker, Zod, Vitest, Testing Library.
